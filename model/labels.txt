# Label Format Notes
# ==================
# - Format: "Crop Disease" (human-readable; matches diseases.xml <name> values).
# - "Healthy" means no disease.
# - The exact label string is used to look up symptoms/treatment/prevention in
#   diseases.xml, so the two files must match character-for-character.
# - Lines starting with '#' and blank lines are ignored by the app loader.

# Mapping to diseases.xml
# =======================
# android-app/app/src/main/assets/diseases.xml has one <disease> entry per label,
# using the exact label string as the <name>:
#
# <disease>
#     <name>Tomato Early Blight</name>
#     <plant>Tomato</plant>
#     <symptoms>Small brown spots with concentric rings...</symptoms>
#     <treatment>Remove infected leaves...</treatment>
#     <prevention>Rotate crops...</prevention>
# </disease>

# Common Issues and Solutions
# ===========================

# Issue: "Index 5 out of bounds for labels list of size 4"
# Solution: Your model has more output classes than you have labels.
#           Add the missing labels or use a model with matching classes.

# Issue: "Predicted class is always 0 or always the same"
# Solution: Check preprocessing. Input must match training preprocessing exactly:
#           - Resize to correct size (e.g., 224x224)
#           - Normalize pixel values (e.g., divide by 255.0)
#           - Convert to RGB (not BGR)

# Issue: "Disease details are blank in the UI"
# Solution: The predicted label has no matching <name> in diseases.xml.
#           Make sure every label here has a diseases.xml entry with the exact
#           same name.
