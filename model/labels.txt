# Label Format Notes
# ==================
# - Format: {Crop}___{Disease} (three underscores)
# - "healthy" means no disease
# - Some labels have parenthetical notes: Grape___Esca_(Black_Measles)
# - Some have spaces: Corn___Cercospora_leaf_spot Gray_leaf_spot
# - "Background_without_leaves" = model detects non-leaf image

# Common Issues and Solutions
# ===========================

# Issue: "Index 5 out of bounds for labels list of size 4"
# Solution: Your model has more output classes than you have labels.
#           Add missing labels or use a model with fewer classes.

# Issue: "Predicted class is always 0 or always same"
# Solution: Check preprocessing. Input must match training preprocessing exactly:
#           - Resize to correct size (e.g., 224x224)
#           - Normalize pixel values (e.g., divide by 255.0)
#           - Convert to RGB (not BGR)

# Issue: "Disease name shows as 'Tomato___Early_blight' in UI"
# Solution: Format the label for display (see format_disease_name function above)

# Issue: "Some predictions are correct, others are random"
# Solution: This is normal for limited datasets. Document accuracy in test results.
#           Focus on mobile engineering, not model accuracy.

# Issue: "Can't find label for prediction"
# Solution: Verify predicted_class index is within range [0, len(labels)-1]
#           Log the predicted index value for debugging
