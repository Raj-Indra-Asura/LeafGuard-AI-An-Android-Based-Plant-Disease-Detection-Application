## Common Evidence for All Weeks

### Weekly Checklist
Each week should have:
- [ ] Weekly progress report (1 page summary)
- [ ] Git commit history screenshot
- [ ] Code files created/modified this week
- [ ] Meeting minutes with guide (if met)
- [ ] Screenshots of work done
- [ ] Any research/learning notes
- [ ] Challenges faced and solutions
- [ ] Next week plan

### Naming Convention
Use format: `YYYYMMDD_weekXX_description.ext`

Example:
- `20260525_week04_login_screen_code.kt`
- `20260525_week04_login_screenshot.png`
- `20260525_week04_guide_meeting.md`

### Organization Tips
1. Create subfolders: `/code`, `/screenshots`, `/documents`, `/diagrams`
2. Keep a master checklist tracking all evidence
3. Backup weekly to cloud (Google Drive, Dropbox)
4. Review previous week evidence before guide meeting
5. Document learnings and challenges weekly
6. Take screenshots of every significant milestone

### File Types to Collect
- **Code:** .kt, .xml, .py, .ipynb
- **Documents:** .md, .pdf, .docx
- **Images:** .png, .jpg (high resolution)
- **Data:** .json, .xml, .csv, .xlsx
- **Videos:** .mp4 (for demos)
- **Archives:** .zip (for code snapshots)
